# PUBLIC-MANIFEST — the positive allowlist of what may exist in this public repo.
#
# WHY A MANIFEST AND NOT (ONLY) A DENYLIST
# test_no_org_literals.py is a denylist: it refuses a file that contains a known-bad string.
# A denylist FAILS OPEN — it cannot catch what nobody thought to enumerate. The leak that
# actually costs us is not a stray name in an approved file; it is a whole NEW file crossing
# over: an unripe module, an internal design doc, a threat-model note that contains no banned
# literal at all. So the rule here is the other way round: **default deny**. A path in this
# repo that is not matched below fails test_public_manifest.py, and the fix is a deliberate
# line in this file — a decision someone made, not a string nobody happened to ban.
#
# HOW TO ADD SOMETHING
# Add the exact path (or a narrow glob) with the line-comment saying what it is. Prefer exact
# paths over broad globs for anything new: `pm_mesh/*.py` is broad on purpose (the shipped
# same-user engine is public in full), but a new top-level dir must be listed explicitly.
#
# SYNTAX: one fnmatch pattern per line, `#` starts a comment, blank lines ignored.
#         Patterns match repo-relative POSIX paths.
